# 发明专利申请书

## 一种面向口腔三维扫描数据的牙齿实例分割与牙位识别方法及系统

---

## 技术领域

本发明涉及口腔医学影像处理与人工智能技术领域,具体涉及一种面向口腔三维扫描(口扫)数据的牙齿实例分割与牙位自动识别方法及系统,可应用于口腔正畸、种植修复、数字化义齿设计等临床场景。

---

## 背景技术

### 1. 口腔三维扫描技术现状

口腔三维扫描(口扫)技术是现代数字化口腔诊疗的基础,通过光学扫描获取患者牙颌模型的高精度三维点云数据。典型的口扫数据包含数万至数十万点,具有以下特点:

- **数据规模大**: 单个牙颌模型通常包含5万-20万点
- **几何结构复杂**: 包含牙齿、牙龈、牙槽骨等多种解剖结构
- **个体差异显著**: 不同患者的牙齿数量、形态、排列差异大
- **边界模糊**: 牙齿与牙龈的边界区域难以准确界定

### 2. 牙齿分割与牙位识别的临床需求

在口腔临床实践中,需要对口扫数据进行:
- **牙齿实例分割**: 精确分离每颗牙齿
- **牙位编号识别**: 按照FDI(国际牙科联合会)牙位表示法识别每颗牙齿的编号
- **边界精确定位**: 牙龈边界、牙间隙等关键区域的准确分割

FDI牙位表示法将牙齿编号为:
- 上颌右侧: 18-11 (从后到前)
- 上颌左侧: 21-28 (从前到后)
- 下颌左侧: 31-38 (从前到后)
- 下颌右侧: 41-48 (从后到前)

共计32颗牙齿,加上上颌牙龈(33类)和下颌牙龈(34类),共34个类别。

### 3. 现有技术的不足

现有口扫数据处理方法存在以下问题:

1. **人工标注效率低**: 传统方法依赖技师手工分割和标注,单个模型需15-30分钟
2. **边界分割精度差**: 牙齿与牙龈边界、牙间隙等区域分割准确率低
3. **牙位识别困难**: 难以自动识别每颗牙齿的具体编号
4. **大规模数据处理慢**: 传统深度学习方法计算复杂度高,难以实时处理
5. **形状变异适应性差**: 对异常牙列(缺牙、多生牙、错位牙)适应性不足

## 发明内容

### 1. 技术问题

本发明要解决的技术问题是:如何实现对口腔三维扫描数据的自动化、高精度、高效率处理,完成牙齿实例分割与牙位编号识别,特别是提高边界区域的分割精度。

### 2. 技术方案

本发明提出一种面向口腔三维扫描数据的牙齿实例分割与牙位识别方法,包括以下步骤:

#### 步骤S1:口扫数据预处理与曲率计算

**S1.1 点云分组采样**

对输入的原始口扫点云进行分层采样:

```
输入: 原始点云 P = {coord, normals},规模N (50,000-200,000点)
处理:
  1. 最远点采样(FPS)选取G个中心点 (G=16,384)
  2. 对每个中心点,通过K近邻(K=11)选取邻近点构成局部组
输出: 分组后的点云,规模(BxG)xK
```

**S1.2 曲率特征计算**

计算各采样点的局部曲率,用于后续的边界增强:

```
对于每个采样点及其K个邻近点:
  1. 提取邻域法向量: {n_1, n_2, ..., n_K}
  2. 计算法向量夹角: angle_i = 2 x arctan2(||n_0xn_i - ||n_0||xn_i||, ||n_0xn_i + ||n_0||xn_i||)
  3. 计算平均曲率: curvature = mean(angle_1, ..., angle_{K-1})
  4. 生成形状权重: shape_weight = 1 if curvature > 5度 else 0
```

**创新点**: 曲率大的点(>5度)位于牙齿-牙龈边界、牙尖、窝沟等几何特征显著区域,形状权重用于后续损失函数的加权,强化边界区域的学习。

#### 步骤S2:多序空间填充曲线序列化

将三维点云转化为多组一维序列,保持空间局部性:

**S2.1 网格坐标计算**

```
grid_coord = floor(coord_max - coord) / grid_size
```

**S2.2 多种空间填充曲线编码**

采用6种空间填充曲线对点云进行序列化:

| 序号 | 曲线类型 | 说明 |
|------|----------|------|
| 1 | Hilbert | 希尔伯特曲线(正向) |
| 2 | Hilbert-Trans | 希尔伯特曲线(转置) |
| 3 | Z-Trans | Z序曲线(转置) |
| 4 | Z-Reverse | Z序曲线(逆序) |
| 5 | Z | Z序曲线(正向) |
| 6 | Hilbert-Reverse | 希尔伯特曲线(逆序) |

**S2.3 序列化编码**

```
对于每种曲线顺序o:
  code_o = encode(grid_coord, batch, depth, order=o)
  order_o = argsort(code_o)  # 排序索引
  inverse_o = scatter(原始索引, order_o)  # 逆排序索引
```

**创新点**: 多种空间填充曲线从不同遍历角度捕获点云的空间结构,避免单一序的偏差,实现对牙颌数据各向同性的特征学习。

#### 步骤S3:多序残差融合的Mamba特征编码

**S3.1 条件位置编码(CPE)**

采用稀疏卷积实现自适应位置编码:

```
输入: 特征 F in R^(Nx D)
处理:
  1. 稀疏卷积: F' = SubMConv3d(F, kernel_size=5)
  2. 线性投影: F'' = Linear(F')
  3. 层归一化: F''' = LayerNorm(F'')
  4. 残差连接: F_out = F + F'''
输出: 位置感知特征
```

**S3.2 多序Mamba处理与残差融合**

对每种序列化顺序执行状态空间模型处理,并通过残差方式融合:

```
hidden_state = 输入特征
对于每种序列化顺序o (共6种):
  1. 按顺序重排: seq_input = scatter(hidden_state, order_o)
  2. Mamba处理: seq_output = Mamba2(seq_input, seq_idx)
  3. 逆序还原: output_o = scatter(seq_output, inverse_o)
  4. 残差累加: hidden_state = hidden_state + output_o
输出: hidden_state (融合了6种序列化顺序的特征)
```

**创新点**:
- 采用Mamba2状态空间模型,计算复杂度为O(N),适合处理大规模口扫数据
- 多序残差融合机制,将不同遍历顺序的特征以累加方式融合,保持特征的各向同性
- 每种序列的输出直接累加到隐藏状态,实现高效的特征融合

#### 步骤S4:编码器-解码器骨干网络

**S4.1 编码器层级结构**

```
Stage 0: [CPE -> 6xMamba] -> 输出96维,16384点
    |
    v GridPooling(stride=2)
Stage 1: [CPE -> 6xMamba] -> 输出96维,8192点
    |
    v GridPooling(stride=2)
Stage 2: [CPE -> 6xMamba] -> 输出192维,4096点
    |
    v GridPooling(stride=2)
Stage 3: [CPE -> 6xMamba] -> 输出384维,2048点
```

**S4.2 解码器层级结构(带跳跃连接)**

```
Stage 3输出
    |
    v GridUnpooling + Skip(Stage 2)
Stage 2': [CPE -> 6xMamba] -> 输出192维
    |
    v GridUnpooling + Skip(Stage 1)
Stage 1': [CPE -> 6xMamba] -> 输出96维
    |
    v GridUnpooling + Skip(Stage 0)
Stage 0': [CPE -> 6xMamba] -> 输出96维
```

**S4.3 多尺度特征收集**

收集编码器各层特征,用于掩码解码器的多尺度注意力:

```
feats = [feat_stage0, feat_stage1, feat_stage2, feat_stage3]
统一投影到d_model=96维
```

#### 步骤S5:牙位感知的查询生成

**S5.1 可学习查询向量**

```
query = Embedding(num_queries=64, d_model=96)  # 64个查询向量
query_pos = Embedding(num_queries=64, d_model=96)  # 查询位置编码
```

**S5.2 查询与牙位的对应关系**

每个查询通过学习,对应特定位置的牙齿实例或牙龈区域。匈牙利匹配器在训练时自动建立查询与真实牙齿的对应关系。

#### 步骤S6:掩码解码与牙位分类

**S6.1 掩码预测**

```
predicted_class = MLP(query)  # [B, 64, 35] (34类+1无目标类)
predicted_mask = query x memory^T / sqrt(d_model)  # [B, 64, G]
attention_mask = sigmoid(predicted_mask) < 0.5  # 掩码注意力
```

**S6.2 多尺度掩码注意力解码**

```
对于每层解码层 (共6层):
  1. 选择对应尺度的编码器特征: encoder_feat = encoder_hidden_states[layer_idx % 4]
  2. 交叉注意力: query = CrossAttention(query, encoder_feat, mask=attention_mask)
  3. 自注意力: query = SelfAttention(query)
  4. 前馈网络: query = FFN(query)
  5. 更新掩码: predicted_mask, attention_mask = MaskPredictor(query)
```

**S6.3 牙位编号到类别的映射**

```
牙编号 -> 类别ID映射:
  18->8, 17->7, 16->6, 15->5, 14->4, 13->3, 12->2, 11->1  (上颌右)
  28->16, 27->15, 26->14, 25->13, 24->12, 23->11, 22->10, 21->9  (上颌左)
  38->24, 37->23, 36->22, 35->21, 34->20, 33->19, 32->18, 31->17  (下颌左)
  48->32, 47->31, 46->30, 45->29, 44->28, 43->27, 42->26, 41->25  (下颌右)
  上颌牙龈->33, 下颌牙龈->34
```

#### 步骤S7:曲率感知的损失函数

**S7.1 匈牙利匹配**

使用匈牙利算法建立预测与真实标签的对应关系:

```
cost = lambda_class x cost_class + lambda_mask x cost_mask + lambda_dice x cost_dice
matched_indices = HungarianMatcher(cost)
```

**S7.2 多任务损失**

```
L_total = L_class + L_mask + L_dice + L_geo
```

其中:

- **分类损失** (交叉熵):
  ```
  L_class = CrossEntropyLoss(predicted_class, target_class)
  ```

- **掩码损失** (二值交叉熵):
  ```
  L_mask = BCEWithLogitsLoss(predicted_mask, target_mask)
  ```

- **Dice损失**:
  ```
  L_dice = 1 - 2x(pred x target).sum() / (pred.sum() + target.sum())
  ```

- **几何感知损失** (创新点):
  ```
  L_geo = shape_weight x FocalLoss(predicted_mask, target_mask)
  ```
  其中shape_weight为曲率权重,高曲率点(边界区域)权重为1,低曲率点权重为0。

**创新点**: 几何感知损失通过对高曲率区域(牙齿边界、牙尖等几何特征显著区域)加权,强化模型对关键边界的分割能力。

#### 步骤S8:特征传播与最终输出

**S8.1 特征上采样**

将分组点的预测结果传播回原始点云:

```
output_feat = KNNInterpolation(sampled_coord, original_coord, sampled_feat, k=11)
```

**S8.2 输出结果**

```
对于原始点云的每个点:
  - 实例掩码: [num_queries] (各查询的掩码响应)
  - 类别概率: [35] (34类+无目标)
  - 最终类别: argmax(类别概率)
```

### 3. 系统架构

本发明提供的口扫数据处理系统包括:

1. **点云预处理模块**: 实现FPS采样、K近邻分组、曲率计算
2. **多序序列化模块**: 生成6种空间填充曲线序列
3. **Mamba编码器**: 多层条件位置编码与多序Mamba处理
4. **池化/反池化模块**: 实现多尺度特征提取与恢复
5. **Mamba解码器**: 对称的上采样结构与跳跃连接
6. **查询生成模块**: 生成可学习的牙齿实例查询
7. **掩码解码器**: 多尺度掩码注意力预测
8. **牙位映射模块**: 牙位编号到类别的转换
9. **损失计算模块**: 曲率感知的多任务损失

## 具体实施方式

### 实施例:全口牙颌模型分割

#### 1. 数据输入

```python
输入数据:
  - 坐标: (N, 3)  # N 约等于 100,000点
  - 法向量: (N, 3)
  - 标签(训练时): (N,)  # 牙位编号11-48, 0表示牙龈

预处理后:
  - 分组数: G = 16,384
  - 组内点数: K = 11
  - 曲率: (G,)  # 用于生成shape_weight
```

#### 2. 模型配置

```python
配置参数:
  # 输入输出
  in_channels: 8  # coord(3) + normals(3) + curvature(1) + s_o_i(1)
  num_labels: 34  # 32颗牙齿 + 上下牙龈

  # 分组参数
  num_group: 16384
  group_size: 11

  # 网络结构
  enc_depths: (6, 3, 6, 3)
  enc_channels: (96, 96, 192, 384)
  dec_depths: (3, 3, 3)
  dec_channels: (96, 96, 192)
  d_model: 96

  # 查询设置
  num_queries: 64

  # 损失权重
  class_weight: 5.0
  mask_weight: 5.0
  dice_weight: 5.0

  # 多序设置
  order: ["hilbert", "hilbert-trans", "z-trans", "z-reverse", "z", "hilbert-reverse"]
```

#### 3. 处理流程

**步骤1**: 预处理
```python
# 最远点采样
s_idx = fps(coord, offset, num_group=16384)  # 16384个中心点

# K近邻分组
s_n_idx, _ = knn(11, coord, offset, s_coord, s_offset)  # 每组11个邻居

# 曲率计算
curvature = point_curvature(s_normals, s_neighbor_normals)
shape_weight = (curvature > 0.087).float()  # 5度阈值
```

**步骤2**: 编码器处理
```python
# 特征编码
feat = MLP(coord, normals, curvature)  # (BG, 96)

# 编码器各层
for stage in encoder_stages:
    feat = CPE(feat)  # 条件位置编码
    feat = MultiOrderMamba(feat, orders)  # 多序Mamba处理
    feat = GridPooling(feat)  # 下采样
```

**步骤3**: 解码器处理
```python
# 解码器各层
for stage in decoder_stages:
    feat = GridUnpooling(feat, skip_feat)  # 上采样+跳跃连接
    feat = CPE(feat)
    feat = MultiOrderMamba(feat, orders)
```

**步骤4**: 掩码预测
```python
# 查询生成
query = query_embedding.weight  # (64, 96)

# 掩码解码
for layer in decoder_layers:
    mask, cls = MaskPredictor(query, memory)
    query = DecoderLayer(query, encoder_feat, mask)

# 输出
pred_mask: (B, 64, G)  # 64个实例掩码
pred_class: (B, 64, 35)  # 35类概率
```

**步骤5**: 损失计算
```python
# 转换标签格式
class_labels, mask_labels = tooth_labels(original_labels, shape_weight)

# 匈牙利匹配
indices = HungarianMatcher(pred_mask, pred_class, mask_labels, class_labels)

# 损失计算
loss = (
    class_weight * CrossEntropyLoss(pred_class, class_labels, indices) +
    mask_weight * BCELoss(pred_mask, mask_labels, indices) +
    dice_weight * DiceLoss(pred_mask, mask_labels, indices) +
    geo_loss(pred_mask, mask_labels, shape_weight)  # 曲率感知损失
)
```
